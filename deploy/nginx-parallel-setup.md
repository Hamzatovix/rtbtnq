# Настройка второго сайта параллельно с первым

## Важно: Первый проект НЕ трогаем!

Если первый проект уже работает на порту 80 через Nginx, мы просто добавляем конфигурацию для второго домена.

---

## Шаг 1: Проверка текущей ситуации

Выполни на сервере:

```bash
# 1. Проверь, что занимает порт 80
netstat -tulpn | grep :80

# 2. Проверь, запущен ли Nginx
systemctl status nginx

# 3. Посмотри существующие конфигурации
ls -la /etc/nginx/sites-enabled/
cat /etc/nginx/sites-enabled/*
```

**Что мы ищем:**
- Если видишь `nginx` в выводе `netstat` → Nginx уже работает, отлично!
- Если видишь конфигурации в `/etc/nginx/sites-enabled/` → Nginx настроен

---

## Шаг 2: Проверка конфигурации для rosebotanique.store

```bash
# Проверь, что конфигурация создана
cat /etc/nginx/sites-available/rosebotanique.store

# Проверь, что она активирована
ls -la /etc/nginx/sites-enabled/ | grep rosebotanique
```

Если конфигурация уже есть и активирована → переходи к Шагу 3.

---

## Шаг 3: Перезагрузка Nginx (БЕЗ остановки!)

```bash
# 1. Проверь конфигурацию на ошибки
nginx -t

# 2. Если всё ОК, перезагрузи Nginx (это НЕ остановит первый сайт!)
systemctl reload nginx

# 3. Проверь статус
systemctl status nginx
```

**Важно:** `reload` не останавливает Nginx, только обновляет конфигурацию!

---

## Шаг 4: Проверка работы обоих сайтов

```bash
# Проверь, что оба сайта работают
curl -H "Host: rosebotanique.store" http://localhost
curl -H "Host: <домен-первого-проекта>" http://localhost
```

---

## Если Nginx НЕ запущен (редкий случай)

Если первый проект работает НЕ через Nginx (например, напрямую через Docker на порту 80), то:

### Вариант A: Использовать другой порт для второго сайта (не рекомендуется)

Можно запустить второй сайт на порту 8080, но тогда нужен другой домен или поддомен.

### Вариант B: Настроить Nginx как основной прокси (требует остановки первого проекта)

Это требует остановки первого проекта, что ты не хочешь делать.

---

## Рекомендуемое решение

**Если первый проект уже использует Nginx:**
1. Конфигурация для `rosebotanique.store` уже создана ✅
2. Просто перезагрузи Nginx: `systemctl reload nginx` ✅
3. Оба сайта будут работать параллельно ✅

**Если первый проект НЕ использует Nginx:**
Нужно узнать больше о первом проекте, чтобы предложить решение без его остановки.
